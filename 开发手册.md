## 2. 后端（摘要）

- **配置**：`SmartVillages-Backend/service/src/main/resources/application.yml`（数据源、MyBatis-Plus、Knife4j、`jwt.secret` / `jwt.expiration-ms` 等）。
- **SQL**：`SmartVillages-Backend/service/src/main/resources/sql/`（如 `auth.sql`、`admin.sql`）；改表同步改 Entity 与脚本。
- **启动**（在 `SmartVillages-Backend` 目录下）：

```bash
./mvnw -pl service -am spring-boot:run
```

- **打包**：

```bash
./mvnw clean package -DskipTests -pl service -am
```

- 多模块修改 `common` 或其它被依赖模块后，若运行结果仍像旧代码，先在 `SmartVillages-Backend` 下执行 **`./mvnw install -DskipTests`**（或至少 `install` 改动的模块）。

## 3. 登录与鉴权（与当前实现一致）

以下为摘要，完整说明见 **`SmartVillages-Backend/README.md` →「登录与鉴权（当前实现）」**。

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/auth/login` | JSON：`username`、`password`；成功返回 `data.token` 等 |
| `DELETE` | `/auth/logout` | 登出 v1（无状态；需请求头 `token`） |
| `GET` | `/admin/me` | 当前用户；`/admin/**` 仅 **`role=admin`** |

- 请求头：**`token`**（JWT）。
- JWT **`subject`**：`id:username:role`；拦截器写入 `authId`、`username`、`role`，可用 **`LoginUserContext`**（`common/context`）读取。
- **两层**：`JwtAuthenticationFilter`（非白名单需 token，否则 **401**）→ `AuthInterceptor`（解析 subject；访问 `/admin` 非 admin → **403**，业务码 **2001**）。
- **白名单**（含登录与文档）：`/auth/login`、`/doc.html`、`/v3/api-docs`、`/swagger-ui` 与 `/swagger-ui/**`、`/webjars` 与 `/webjars/**`、`OPTIONS`。
- Knife4j：**`/doc.html`**；OpenAPI：**`/v3/api-docs`**（须在白名单内，否则无 token 会 **401**）。

## 4. 统一返回与全局异常（要点）

- **`ErrorCode`**：`1xxx` 认证用户，`2xxx` 权限，`3xxx` 文件，`4xxx` 业务，`5xxx` 系统，`6xxx` 路由语义；同一语义唯一数字。
- **`Result`**：字段 **`code` / `message` / `data`**；成功常用 **`code = 200`**（见 `Result.SUCCESS_CODE`）。Controller 中多用 **`Result.success(...)`** / **`Result.fail(...)`** / **`Result.error(...)`**（以源码为准）。
- **`GlobalExceptionHandler`**：已进入 MVC 的异常统一转 `Result`。**`BusinessException(NO_PERMISSION)`** → **HTTP 403**，业务码 **2001**。过滤器阶段 token 失败多为 **401**，通常**无** `Result` JSON。

## 5. Maven 多模块约定

- 父工程：`SmartVillages-Backend/pom.xml`（`dependencyManagement`、`<modules>`）。
- **`common`** 不依赖任何业务模块；业务模块只依赖 **`common`**。
- **`service`** 必须依赖 **`common` + 全部业务子模块**，否则对应包不会进运行时。
- **不要在手册里粘贴易过时的完整 `pom` 片段**；依赖与版本以仓库内 **`SmartVillages-Backend/common/pom.xml`**、**`SmartVillages-Backend/service/pom.xml`**、父 `pom` 为准（当前 Web 使用 **`spring-boot-starter-web`**，Knife4j 以父工程管理的社区 fork 版本为准）。

新增业务子模块时：父 `pom` 增加 `<module>`，**`service/pom.xml` 增加对该模块的依赖**，再 `mvn install`。

## 6. 接口文档（Knife4j / OpenAPI）

- 依赖集中在后端 **`common`**（Knife4j OpenAPI3 Jakarta starter + SpringDoc，具体以 `common/pom.xml` 为准）。
- 浏览器访问：`http://localhost:<端口>/doc.html`；OpenAPI JSON：`/v3/api-docs`。
- Controller 使用 **`@Tag`、`@Operation`**（`io.swagger.v3.oas.annotations`）。

## 7. 前端

- **目录**：`SmartVillages-Web/`

```bash
cd SmartVillages-Web
npm install
npm run dev
```

```bash
npm run build
```

- **联调**：用 `.env`（如 `VITE_API_BASE_URL`）配置后端地址；请求头携带后端约定的 **`token`**。跨域任选其一：**Vite proxy** 或后端 **CORS**，避免重复叠加。

## 8. 本地环境

- JDK **17**、本机 **MySQL** 与 `application.yml` 中库名/账号一致后再启动后端。
