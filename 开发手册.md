## 2. 后端（摘要）

- **配置**：`SmartVillages-Backend/service/src/main/resources/application.yml`（数据源、MyBatis-Plus、Knife4j、`jwt.secret` / `jwt.expiration-ms`、Redis 等）。
- **SQL**：`SmartVillages-Backend/service/src/main/resources/sql/`（如 `auth.sql`、`admin.sql`）；改表同步改 Entity 与脚本。
- **本地依赖**：除 MySQL 外，公告详情缓存等使用 **Redis**（默认见 `application.yml`）。
- **启动**（在 `SmartVillages-Backend` 目录下）：

```bash
./mvnw -pl service -am spring-boot:run
```

- **打包**：

```bash
./mvnw clean package -DskipTests -pl service -am
```

- 多模块修改 `common` 或其它被依赖模块后，若运行结果仍像旧代码，先在 `SmartVillages-Backend` 下执行 **`./mvnw install -DskipTests`**（或至少 `install` 改动的模块）。

## 3. 登录与鉴权（与当前实现一致）

以下为摘要，完整说明见 **`SmartVillages-Backend/README.md` →「登录与鉴权（当前实现）」**。

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/auth/login` | JSON：`username`、`password`；成功返回 `data.token` |
| `DELETE` | `/auth/logout` | 登出 v1（无状态；建议仍带 `token`，语义与前端约定一致即可） |
| `GET` | `/admin/me` | 当前用户；需 **管理员** 权限（`ROLE_ADMIN`） |

- 请求头：**`token`**（JWT）。
- JWT **`subject`**：`id:username:role`；**`JwtSecurityFilter`** 解析后：  
  - 写入 **`SecurityContext`**（`GrantedAuthority` 形如 **`ROLE_ADMIN` / `ROLE_CADRE` / `ROLE_VILLAGER`**，由角色字符串规范化而来）；  
  - 同步写入 request attribute：`authId`、`username`、`role`，便于 **`LoginUserContext`** 读取。
- **授权模型**：以 **`SecurityConfig`** 中的 **`authorizeHttpRequests`** 为准（`permitAll` 白名单、`hasAuthority` / `hasAnyAuthority` 与 `authenticated()`）。未认证访问受保护资源时由 **Spring Security** 返回 **401**；已认证但权限不足返回 **403**（与旧版「过滤器 401 + 拦截器 403」拆分方式不同，以运行时行为为准）。
- **白名单（节选，完整列表见后端 README）**：`/auth/login`、`/doc.html`、`/v3/api-docs/**`、`/swagger-ui/**`、`/webjars/**`、`/swagger-resources/**`、以及业务上公开的 **`/announcements/**`**、**`/interactions/**`**、预留的 **`/guest/**`** 等。
- Knife4j：**`/doc.html`**；OpenAPI：**`/v3/api-docs`**（具体子路径与版本以后端依赖为准）。

## 4. 统一返回与全局异常（要点）

- **`ErrorCode`**：`1xxx` 认证用户，`2xxx` 权限，`3xxx` 文件，`4xxx` 业务，`5xxx` 系统，`6xxx` 路由语义；同一语义唯一数字。
- **`Result`**：字段 **`code` / `message` / `data`**；成功常用 **`code = 200`**（见 `Result.SUCCESS_CODE`）。Controller 中多用 **`Result.success(...)`** / **`Result.fail(...)`** / **`Result.error(...)`**（以源码为准）。
- **`GlobalExceptionHandler`**：已进入 MVC 的异常统一转 `Result`。Spring Security 在过滤器链上产生的 **401/403** 多为标准响应，**不一定**带 `Result` JSON，联调时请以 HTTP 状态与响应体为准。

## 5. Maven 多模块约定

- 父工程：`SmartVillages-Backend/pom.xml`（`dependencyManagement`、`<modules>`）。
- **`common`** 不依赖任何业务模块；业务模块只依赖 **`common`**。
- **`service`** 必须依赖 **`common` + 全部业务子模块**，否则对应包不会进运行时。
- **不要在手册里粘贴易过时的完整 `pom` 片段**；依赖与版本以仓库内 **`SmartVillages-Backend/common/pom.xml`**、**`SmartVillages-Backend/service/pom.xml`**、父 `pom` 为准（当前含 **`spring-boot-starter-security`**、**`spring-boot-starter-data-redis`**、Knife4j / SpringDoc 等）。

新增业务子模块时：父 `pom` 增加 `<module>`，**`service/pom.xml` 增加对该模块的依赖**，再 `mvn install`。

## 6. 接口文档（Knife4j / OpenAPI）

- 依赖集中在后端 **`common`**（具体以 `common/pom.xml` 为准）。
- 浏览器访问：`http://localhost:<端口>/doc.html`；OpenAPI JSON：`/v3/api-docs`（或带后缀的变体，以实际暴露路径为准）。
- Controller 使用 **`@Tag`、`@Operation`**（`io.swagger.v3.oas.annotations`）。

## 7. 前端

- **目录**：`SmartVillages-Web/`
- **现状**：见 **`SmartVillages-Web/README.md`**（当前无 `vue-router` / `pinia` / axios 依赖，以仓库 `package.json` 为准）。

```bash
cd SmartVillages-Web
npm install
npm run dev
```

```bash
npm run build
```

- **后续联调建议**：用环境变量（如 Vite 的 `import.meta.env`）配置后端 base URL；请求头携带 **`token`**。跨域任选其一：**Vite proxy** 或后端 **CORS**，避免重复叠加。

## 8. 本地环境

- JDK **17**、本机 **MySQL** 与 `application.yml` 中库名/账号一致。
- **Redis**（若使用默认配置）：与 `application.yml` 中 host/port/password 一致；公告等模块会用到缓存工具类。

## 9. 当前进度与下一步（建议）

### 当前进度

- 后端：**Spring Security + JWT** 已接入；管理员用户分页、状态变更、创建村干部账号等与公告/留言相关接口已有多处实现（以后端源码与 Knife4j 为准）。
- 文档：本仓库 README / 开发手册与后端 README、后端开发手册已按当前主干实现对齐到「章节级」；细粒度字段仍以代码与 OpenAPI 为准。
- 前端：页面与布局文件存在，**应用入口尚未挂载路由与接口层**，需按前端 README 分步补齐后再做端到端联调。

### 下一步（可按优先级调整）

1. 前端：引入 **`vue-router` + `pinia`**，装配 `App.vue` / Layout，与后端 **`token`** 约定打通。
2. 前端：抽 **`shared/api`**（axios 或 fetch）与 **`services/*.api.js`**，对齐 `Result` 结构。
3. 后端：按业务继续补 **村务 management**、**媒体 media** 等模块接口与验收清单。
4. 运维向：`jwt.secret`、数据库与 Redis 密码等改为 **环境变量或独立 profile**，避免提交生产密钥。
