2.**可复用项目架构**：Maven 多模块 + 根包 `com.backend.*`（单 Spring Boot 进程，由 `service` 启动）。复制到新项目时保留「模块边界」与下面「包—职责」对应关系即可。

**依赖与职责约定**

| 位置 | Maven 依赖 | 主要内容 |
|------|------------|----------|
| 父工程 `smartVillages/` | 无业务依赖 | `pom.xml`：`<modules>`、`dependencyManagement`（第三方版本钉死） |
| `common` | 不写对业务模块的依赖 | Web、MyBatis-Plus、Knife4j(OpenAPI3/Jakarta)、JWT 等；`config` / `exception` / `filter` / `result` / `utils` |
| 各业务子模块（如 `auth`、`admin`…） | 仅依赖 `common`（及 Lombok 等） | 领域代码，包名 `com.backend.<模块名>` |
| `service` | 依赖 `common` + 所有业务模块 + MySQL 驱动 | `SmartVillagesApplication`、`application.yml`、`resources/sql/`、集成测试 |

**仓库目录与公共包（可直接照抄）**

```text
smartVillages/
├── pom.xml
└── common/
    └── src/main/java/com/backend/common/
```

**各模块 `pom.xml` 依赖怎么写（可直接套模板）**

- 版本号：业务相关的第三方（MyBatis-Plus、Knife4j、jjwt 等）放在**父工程**的 `<dependencyManagement>` 里统一写 `<version>`；子模块里**不写版本**，只写 `<groupId>` + `<artifactId>`。
- 内部模块：一律 `<version>${project.version}</version>`，与父工程 `0.0.1-SNAPSHOT` 对齐。

**common**（公共能力，不要依赖任何业务 `artifactId`）

```xml
<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-webmvc</artifactId>
    </dependency>
    <dependency>
        <groupId>com.baomidou</groupId>
        <artifactId>mybatis-plus-spring-boot3-starter</artifactId>
    </dependency>
    <dependency>
        <groupId>com.github.xiaoymin</groupId>
        <artifactId>knife4j-openapi3-jakarta-spring-boot-starter</artifactId>
    </dependency>
    <dependency>
        <groupId>io.jsonwebtoken</groupId>
        <artifactId>jjwt</artifactId>
    </dependency>
    <dependency>
        <groupId>javax.xml.bind</groupId>
        <artifactId>jaxb-api</artifactId>
    </dependency>
    <dependency>
        <groupId>org.projectlombok</groupId>
        <artifactId>lombok</artifactId>
        <optional>true</optional>
    </dependency>
</dependencies>
```

**业务子模块**（以 `auth` 为例；`admin`、`announcement` 等只改 `<artifactId>auth</artifactId>` 与目录名）

```xml
<dependencies>
    <dependency>
        <groupId>com.backend</groupId>
        <artifactId>common</artifactId>
        <version>${project.version}</version>
    </dependency>
    <dependency>
        <groupId>org.projectlombok</groupId>
        <artifactId>lombok</artifactId>
        <optional>true</optional>
    </dependency>
</dependencies>
```

**service**（启动模块）：`<dependencies>` 里聚合 **common**、**当前全部业务子模块**（各一条 `com.backend` + `${project.version}`），再加 `spring-boot-starter-webmvc`、`mysql-connector-j`（`runtime`）、`spring-boot-starter-test`（`test`）；`<build>` 里配置 `spring-boot-maven-plugin`。**完整写法直接看仓库里的 `service/pom.xml`，不在手册里重复粘贴，避免以后要改两处。**

新增业务模块时：父 `pom` 的 `<modules>` 增一行，**`service/pom.xml` 的 `<dependencies>` 增一条同模块**，再根目录 `mvn install`。

3.分完包，就对子模块的依赖版本进行控制，下面是必要的父依赖填写信息

```
<parent>
    <groupId>com.backend</groupId>
    <artifactId>smartVillages</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    <relativePath>../pom.xml</relativePath>
</parent>
```

4. **`service/pom.xml`** 必须带上所有业务模块依赖，漏了哪个，运行时就不会加载那个包下的 Bean。若只改了 `common`，建议根目录 `mvn install`（或至少 `install common`）再运行/打包。

5.根目录不要再保留旧的 `src/main`；`application.yml` 与 `sql` 只放在 `service/src/main/resources/`。
