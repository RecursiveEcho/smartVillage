# 智慧乡村后端开发手册

## 1. 目标与原则

- 本手册用于指导 `SmartVillages-Backend` 的日常开发与排障。
- 技术栈：Spring Boot 3.3.x + **Spring Security** + MyBatis-Plus + Knife4j(OpenAPI3) + JWT + MySQL + **Redis**（公告等模块使用缓存工具类）。
- 原则：先完成最小可用闭环，再逐步增强；每项功能需可验证。

## 2. 项目结构与模块职责

```text
SmartVillages-Backend/
├── pom.xml
├── common/          # 公共能力（不依赖任何业务模块）
├── auth/            # 登录认证
├── admin/           # 管理员相关接口
├── announcement/    
├── management/
├── feature/
├── interaction/
├── media/
└── service/         # 启动模块（聚合所有业务模块）
```

### 模块依赖规则

- `common` 只能放通用能力：`config`、`exception`、`filter`、`result`、`utils`、`enums`、`context` 等。
- 业务模块（如 `auth`、`admin`）只依赖 `common`。
- `service` 必须依赖 `common + 全部业务模块`，否则运行时会缺 Bean。

## 3. 关键配置约定

### 3.1 运行配置

- 统一配置文件：`service/src/main/resources/application.yml`。
- SQL 脚本目录：`service/src/main/resources/sql/`。
- 本地调试建议使用 `application-dev.yml` 或环境变量，避免生产配置进仓库。

### 3.2 OpenAPI / Knife4j

- 文档地址：`http://localhost:8080/doc.html`（端口以实际启动日志为准）。
- OpenAPI 地址：`http://localhost:8080/v3/api-docs`（或带 `/**` 子路径，以依赖版本为准）。
- 匿名可访问路径以 `SecurityConfig` 中 `permitAll` 的 `requestMatchers` 为准（含 `/auth/login`、文档路径、`/announcements/**`、`/interactions/**`、预留 `/guest/**` 等）。
- 不单独维护与代码脱节的白名单文档。

### 3.3 Redis

- 默认连接信息见 `application.yml` 中 `spring.data.redis`。
- `RedisJsonCacheTool`、`AnnouncementServiceImpl` 等会访问 Redis；本地未启动 Redis 时，需调整配置或自动装配策略，否则可能启动失败或运行时报错。

## 4. 统一返回与错误处理

- 错误码枚举：`com.backend.common.enums.ErrorCode`
  - `1xxx` 认证用户
  - `2xxx` 权限角色
  - `3xxx` 文件资源
  - `4xxx` 通用业务
  - `5xxx` 系统异常
  - `6xxx` 路由/HTTP 语义
- 统一返回：`com.backend.common.result.Result`
- 全局异常：`com.backend.common.exception.GlobalExceptionHandler`
  - 已进入 MVC 的业务异常统一转 `Result`
  - **未登录 / 无权限**：优先以 **Spring Security** 返回的 **401 / 403** 为准；响应体不一定是 `Result` 结构

## 5. 登录模块当前基线（已落地）

当前认证与授权链路要点：

1. `POST /auth/login`
   - 校验账号密码
   - 校验删除状态、禁用状态
   - 生成 JWT（subject：`id:username:role`）
2. **`JwtSecurityFilter`**
   - 读取请求头 **`token`**
   - 解析成功：写入 **`SecurityContext`**（`ROLE_*`）与 **request attribute**（`authId`、`username`、`role`）
3. **`SecurityConfig`**
   - `permitAll` / `hasAuthority` / `hasAnyAuthority` / `authenticated()` 组合控制接口访问
4. `GET /admin/me`
   - 返回当前登录主体（实现以 `AdminController` 为准）
5. `DELETE /auth/logout`（v1）
   - 无状态退出：后端返回成功，前端清理本地 token

### 管理员域（已落地，节选）

- `GET /admin/users`：分页 + 可选筛选（`username`、`role`、`status` 等，以 Controller 为准）
- `PUT /admin/users/{id}/status`：启用/禁用
- `POST /admin/cadre`：创建村干部（`AuthDTO` + 校验）

## 6. Auth 模块开发流程

### 第一步：数据库

- 确认库名与 `application.yml` 一致。
- 导入 `sql/auth.sql`、`sql/admin.sql` 等。
- 先验证是否能查到管理员账号。

### 第二步：实体与 Mapper

- `AuthEntity` 必须准确映射 `auth` 表。
- 若逻辑删除列是 `is_deleted`，实体字段需做列名映射。
- `AuthMapper` 使用 `@Mapper` + `BaseMapper<AuthEntity>`。

### 第三步：服务层（登录闭环）

- 非空校验
- 按 `username` 查询用户
- 校验删除/禁用
- MD5（32 位小写十六进制）比对（与库中存证一致）
- 返回 `JwtResponse`（当前以 **`token`** 字段为主）

### 第四步：控制层

- `@RequestBody LoginRequest` 处理 JSON 登录参数
- `DELETE /auth/logout`

### 第五步：鉴权与权限

- **新增 URL 前缀时**：同时更新 **`SecurityConfig`** 的 `requestMatchers`，并确认 `JwtSecurityFilter` 是否仍需为业务提供 `LoginUserContext` 所需的 attribute。
- **角色与权限**：数据库中的 `role` 字符串应能与 **`JwtSecurityFilter.normalizeRole`** 生成的 **`ROLE_*`** 对齐。

### 第六步：验收（示例）

- 未带 token 访问需认证的接口：应 **401**（Spring Security 默认行为）
- `ROLE_ADMIN` 访问 `GET /admin/me`：应 `200` 且 `Result` 成功
- 非管理员访问仅管理员可访问的路径：应 **403**
- 访问 `/doc.html`：应在 `permitAll` 下返回 **200**

## 7. 当前进度与下一步（后端）

### 当前进度

- **Spring Security** + **JWT** 已接入，`JwtSecurityFilter` 与 `SecurityConfig` 为鉴权主路径。
- 管理员用户分页、状态维护、创建村干部等已在 `admin` 模块落地。
- 公告、留言等模块已有主要接口；公告使用 **Redis** 做详情缓存。

### 下一步

1. 为关键路径补 **集成测试**（含 Security 规则回归）。
2. 完善 **management / media / feature** 等与前端页面匹配的 API。
3. 将敏感配置迁移至 **环境变量** 或多 profile。

### 本阶段常见坑

- 登录状态判断条件用 `&&`，不要写成 `||`。
- 分页需启用 MyBatis-Plus 分页插件，且依赖里包含 `mybatis-plus-jsqlparser`。
- 改了 `common` 后服务行为不更新时，先执行 `mvn install -DskipTests` 再重启。
- **只改 Controller 路径却忘记改 `SecurityConfig`**：表现为 401/403 与预期不符。

## 8. 本地运行与打包

### 启动

```bash
cd SmartVillages-Backend
./mvnw -pl service -am spring-boot:run
```

### 打包

```bash
cd SmartVillages-Backend
./mvnw clean package -DskipTests -pl service -am
java -jar service/target/service-0.0.1-SNAPSHOT.jar
```

### 多模块注意事项

- 改了 `common` 或任一被依赖模块后，优先执行一次根目录 `mvn install`。
- 若接口改了但运行结果没变，通常是本地依赖 jar 未更新，重新 `install` 后再启动。

### 媒体上传接口契约（V1）

#### 1）接口用途
用于上传村干部头像图片到 OSS，返回可展示的图片地址。

#### 2）接口定义
- 路径：`POST /media/images/upload`
- Content-Type：`multipart/form-data`
- 鉴权：需要登录；角色仅允许 `ROLE_ADMIN`、`ROLE_CADRE`

#### 3）请求参数
- `file`（必填）：图片文件
- `bizType`（必填）：业务类型，V1 只允许 `avatar`
