# 智慧乡村后端开发手册（执行版）

## 1. 目标与原则

- 本手册用于指导 `SmartVillages-Backend` 的日常开发与排障。
- 技术栈：Spring Boot 3.3.x + MyBatis-Plus + Knife4j(OpenAPI3) + JWT + MySQL。
- 原则：先做最小闭环，再做增强；每个功能必须可验证再进入下一步。

## 2. 项目结构与模块职责

```text
SmartVillages-Backend/
├── pom.xml
├── common/          # 公共能力（不依赖任何业务模块）
├── auth/            # 登录认证
├── admin/           # 管理员相关接口
├── announcement/
├── management/
├── feature/
├── interaction/
├── media/
└── service/         # 启动模块（聚合所有业务模块）
```

### 模块依赖规则

- `common` 只能放通用能力：`config`、`exception`、`filter/interceptor`、`result`、`utils`、`enums`、`context`。
- 业务模块（如 `auth`、`admin`）只依赖 `common`。
- `service` 必须依赖 `common + 全部业务模块`，否则运行时会缺 Bean。

## 3. 关键配置约定

### 3.1 运行配置

- 统一配置文件：`service/src/main/resources/application.yml`。
- SQL 脚本目录：`service/src/main/resources/sql/`。
- 本地调试建议使用 `application-dev.yml` 或环境变量，避免生产配置进仓库。

### 3.2 OpenAPI / Knife4j

- 文档地址：`http://localhost:8080/doc.html`。
- OpenAPI 地址：`http://localhost:8080/v3/api-docs`。
- 常见白名单（JWT 放行）：
  - `/auth/login`
  - `/doc.html`
  - `/v3/api-docs`
  - `/swagger-ui/**`
  - `/webjars/**`

## 4. 统一返回与错误处理

- 错误码枚举：`com.backend.common.enums.ErrorCode`
  - `1xxx` 认证用户
  - `2xxx` 权限角色
  - `3xxx` 文件资源
  - `4xxx` 通用业务
  - `5xxx` 系统异常
  - `6xxx` 路由/HTTP 语义
- 统一返回：`com.backend.common.result.Result`
- 全局异常：`com.backend.common.exception.GlobalExceptionHandler`
  - `NO_PERMISSION` 统一映射为 `HTTP 403`
  - token 缺失/无效由过滤器或拦截器处理，通常返回 `HTTP 401`

## 5. 登录模块当前基线（已落地）

当前认证链路已具备以下能力：

1. `POST /auth/login`
   - 校验账号密码
   - 校验删除状态、禁用状态
   - 生成 JWT（subject：`id:username:role`）
2. 鉴权拦截
   - 校验 `token` 请求头
   - 拦截后写入请求上下文：`authId`、`username`、`role`
3. 角色控制
   - `/admin/**` 仅允许 `role=admin`
   - 非 admin 返回 `403 + code=2001`
4. `GET /admin/me`
   - 返回当前登录用户信息（含角色）
5. `POST /auth/logout`（v1）
   - 无状态退出：后端返回成功，前端清理本地 token

## 6. Auth 模块开发流程（可重复执行）

### 第一步：数据库

- 确认库名与 `application.yml` 一致。
- 导入 `sql/auth.sql`、`sql/admin.sql`。
- 先验证是否能查到管理员账号。

### 第二步：实体与 Mapper

- `AuthEntity` 必须准确映射 `auth` 表。
- 若逻辑删除列是 `is_deleted`，实体字段需做列名映射。
- `AuthMapper` 使用 `@Mapper` + `BaseMapper<AuthEntity>`。

### 第三步：服务层（登录闭环）

- 非空校验
- 按 `username` 查询用户
- 校验删除/禁用
- MD5（32位小写）比对
- 返回 `JwtResponse`（含 token）

### 第四步：控制层

- `@RequestBody LoginRequest` 处理 JSON 登录参数
- 新增 `logout` 接口（v1）

### 第五步：鉴权与权限

- 过滤器和拦截器都要放行文档地址，避免 Knife4j 401。
- 拦截器解析 token subject，并写入 request attribute。
- `/admin/**` 做角色判断，不足权限抛业务异常。

### 第六步：验收

- 未带 token 访问 `/admin/me`：应返回 `401`
- admin token 访问 `/admin/me`：应返回 `200`
- villager token 访问 `/admin/me`：应返回 `403 + 2001`
- 访问 `/doc.html`：应返回 `200`

## 7. 当前进度与下一步（后端）

### 当前进度

- 认证闭环已完成：`/auth/login`、`/auth/logout`、`/admin/me`。
- 鉴权语义已对齐：token 无效/缺失 `401`，角色不足 `403 + 2001`。
- 文档页已放行：`/doc.html`、`/v3/api-docs`。

### 下一步（最小功能）

- 先做：`GET /admin/users` 分页查询（仅 admin，返回 VO，不返回密码）。
- 再做：用户名/角色/状态筛选。
- 最后做：账号启用/禁用接口与联调验收。

### 本阶段常见坑

- 登录状态判断条件用 `&&`，不要写成 `||`。
- 分页配置需启用 MyBatis-Plus 分页插件，且模块依赖里要包含 `mybatis-plus-jsqlparser`。
- 改了 `common` 后服务行为不更新时，先执行 `mvn install -DskipTests` 再重启。

## 8. 本地运行与打包

### 启动

```bash
cd SmartVillages-Backend
./mvnw -pl service -am spring-boot:run
```

### 打包

```bash
cd SmartVillages-Backend
./mvnw clean package -DskipTests -pl service -am
java -jar service/target/service-0.0.1-SNAPSHOT.jar
```

### 多模块注意事项

- 改了 `common` 或任一被依赖模块后，优先执行一次根目录 `mvn install`。
- 若接口改了但运行结果没变，通常是本地依赖 jar 未更新，重新 `install` 后再启动。
